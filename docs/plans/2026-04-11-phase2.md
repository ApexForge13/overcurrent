## Task 1: Fact Survival Tracking in Synthesis Agent

**Files:**
- Modify: `src/agents/synthesis.ts`

Add to the synthesis prompt (alongside buried_evidence), a `fact_survival` section that tracks which facts survived each editorial boundary:

Add to system prompt:
```
FACT SURVIVAL TRACKING:

For the most important 3-5 facts in this story, trace their survival
through editorial layers:

Layer 1: On-scene / primary source (eyewitness, official statement, document)
Layer 2: Local media (local TV, local newspaper)
Layer 3: National media (national TV, national newspaper, wire services)
Layer 4: International media (foreign outlets covering the story)

For each fact, note:
- Which layer it originated at
- Which layers it survived to
- Where it DIED (which boundary filtered it out)
- What was lost when it died (context, nuance, counter-narrative)

A fact "dies" when it appears in one layer but not the next.
Not all fact deaths are significant — routine details naturally
get filtered. Flag only facts whose death changes the narrative.
```

Add to JSON response shape:
```json
"fact_survival": [
  {
    "fact": "Coworker said 'I was making good money'",
    "origin_layer": "local",
    "survived_to": "local",
    "died_at": "national",
    "kill_point": "CBS LA → national outlets",
    "what_was_lost": "The only on-record counter-narrative to wage claims",
    "significance": "HIGH | MEDIUM | LOW",
    "sort_order": 1
  }
]
```

Add to SynthesisResult interface:
```typescript
factSurvival: Array<{
  fact: string
  originLayer: string
  survivedTo: string
  diedAt: string
  killPoint: string
  whatWasLost: string
  significance: string
  sortOrder: number
}>
```

Add parsing with snake_case→camelCase conversion and safe defaults.
